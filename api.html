Environment

aamaslib.ca.envs

Generic Cellular Automaton environment that is rule-agnostic and accepts pluggable update rules. Designed to be simple to use in AAMAS worksheets.

Overview

  • Rule-agnostic: you can plug any cell-update function.

  • Neighbourhoods and boundary handling use the utilities provided by aamaslib.ca.

  • Useful for small experiments and educational purposes.

Example

Basic usage with a simple “Game of Life”-like rule:

import numpy as np from aamaslib.ca import moore, boundary_dead, GridRenderer from aamaslib.ca.envs import CellularEnvironment

def life_rule(grid, i, j, neighbours):

# classic Conway if grid[i, j] == 1:

return 1 if neighbours in (2, 3) else 0

else:

return 1 if neighbours == 3 else 0

config = {
“grid”: [

“………………..”, “………………..”, “….OOO………….”, “….O……………”, “…..O…………..”, “………………..”, “………………..”, “………………..”, “………………..”, “………………..”

], “max_iterations”: 50, “neighbourhood”: {“type”: “moore”, “radius”: 1}, “boundary”: {“type”: “dead”}, “rule”: {“type”: “conway”} # used only if you also pass a rule_registry

}

renderer = GridRenderer(mode=”display”, pause_ms=100) env = CellularEnvironment(config, render_mode=”human”, renderer=renderer, update_rule=life_rule)

grid = env.reset() terminated = truncated = False while not (terminated or truncated):

grid, terminated, truncated, info = env.step() env.render()

env.close()

class aamaslib.ca.envs.CellularEnvironment(config: Dict[str, Any], env_shape: str = 'grid', rule_registry: Dict[str, Callable[[ndarray, int, int, int], int]] | None = None, render_mode: str | None = None, renderer: Any | None = None, update_rule: UpdateRuleProtocol | Callable[[ndarray, int, int, int], int] | None = None)[source]

Bases: object

Generic cellular automaton environment with pluggable rules.

This environment evolves a binary grid (0 = dead, 1 = alive) using a provided update rule and configurable neighborhood and boundary conditions.

Parameters:

  • config (dict) –

    Configuration dictionary. Expected keys: - grid: list[str]

    Initial grid as a list of strings where ‘O’ is alive and any other character is treated as dead.

    • max_iterations: int

      Maximum number of steps before truncation.

    • neighbourhood: dict
      {

      “type”: “moore” | “von_neumann” | “custom”, “radius”: int (for moore or von_neumann), “custom_offsets”: list[tuple[int, int]] (if type == “custom”)

      }

    • boundary: dict

      {“type”: “dead” | “wrap” | “mirror”}

    • rule: dict

      {“type”: “<name>”} used only if you also pass a rule_registry.

  • env_shape (str, default="grid") – Currently supports only “grid” (directly uses config[“grid”]).

  • rule_registry (dict[str, Callable], optional) – Optional registry for looking up a rule by name in config[“rule”][“type”]. If provided and no explicit update_rule is passed, the environment will resolve the rule via this registry.

  • render_mode (str, optional) – If “human” and a renderer is provided, render() will display the grid.

  • renderer (Any, optional) – An object compatible with aamaslib.ca.rendering.GridRenderer: must implement .show(grid, iteration, update_rule_name) and .close().

  • update_rule (Callable, optional) – Explicitly provide the rule callable. If provided, it overrides rule_registry.

Notes

  • The environment is intentionally minimal and not tied to Gymnasium APIs, but it follows a similar pattern: reset() and step() returning (observation, terminated, truncated, info).

  • The observation is the current grid (np.ndarray[int]).
